# Contributing to Cosmopilot

Thank you for your interest in contributing to Cosmopilot! We welcome contributions from developers, designers, and enthusiasts.

## How to Contribute

### Reporting Issues

1. Check if the issue already exists in [GitHub Issues](https://github.com/NicoGrassetto/Cosmopilot/issues)
2. If not, create a new issue with a clear title and description
3. Include steps to reproduce bugs and expected vs. actual behavior
4. Use the provided issue template for consistency

### Submitting Code

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes and test thoroughly
4. Commit with clear, descriptive messages
5. Push to your fork
6. Open a pull request with a clear description of changes

### Pull Request Guidelines

- Link related issues in the PR description
- Provide before/after examples for UI changes
- Ensure code follows project conventions
- Add tests for new functionality
- Keep commits clean and focused

## Development Setup

1. Clone the repository
2. Install dependencies:
   - Frontend: `cd frontend && npm install`
   - Infrastructure: Ensure Azure CLI is installed
3. Run tests locally before submitting PR
4. Use Helix editor + Zellij for development

## Code Standards

- Follow existing code style and conventions
- Write clear, commented code
- Test your changes thoroughly
- Update documentation as needed

## Questions?

- Open a discussion in GitHub Discussions
- Check the FAQ in README
- Review existing documentation in `/docs`

Thank you for contributing to Cosmopilot!
