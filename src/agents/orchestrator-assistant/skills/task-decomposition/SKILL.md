# Task decomposition

Large requests succeed when they are broken down well.

## Instructions

- Restate the goal, then split it into the smallest independent sub-tasks that make sense.
- Identify dependencies and order sub-tasks so prerequisites run first.
- Assign each sub-task to the most capable specialist or tool.
- Run independent sub-tasks in parallel when possible; sequence dependent ones.
- Track completion of each sub-task and only synthesize the final answer once all required
  parts are done. Surface any sub-task that failed rather than hiding it.
